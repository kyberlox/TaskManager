# Task Manager – Управление задачами с рекурсивными подзадачами и сообщениями

Веб-приложение для управления задачами, построенное на стеке FastAPI, SQLAlchemy, Jinja2, PostgreSQL, Nginx и Docker. Позволяет создавать задачи с неограниченной вложенностью, общаться внутри задач, загружать файлы, просматривать календарь событий и многое другое.

## 🚀 Особенности

- **Рекурсивные подзадачи** – создавайте задачи внутри задач без ограничений.
- **Встроенный чат** – отправляйте сообщения внутри задачи, прикрепляйте файлы (изображения, видео, PDF, Excel и др.).
- **Календарь** – визуализация задач на временной шкале.
- **Тёмный дизайн** – фиолетово-зелёная цветовая схема, адаптивная вёрстка.
- **Полноценная авторизация** – сессии через куки, регистрация, профиль с аватаркой.
- **Управление категориями** – цветные категории для группировки задач.
- **Загрузка файлов** – превью задач, аватарки пользователей, вложения к сообщениям.
- **Рендеринг файлов** – встроенный просмотр изображений, видео, PDF, Excel таблиц, текстовых файлов прямо в браузере.
- **Административный интерфейс** – для администраторов доступен список всех пользователей.

## 🛠 Технологии

- **Backend**: Python 3.12, FastAPI, SQLAlchemy, Jinja2, Pydantic
- **База данных**: PostgreSQL
- **Frontend**: HTML5, CSS3, JavaScript (Vanilla + FullCalendar)
- **Веб-сервер**: Nginx (обратный прокси, раздача статики)
- **Контейнеризация**: Docker, Docker Compose
- **Аутентификация**: сессии через куки, хеширование паролей (pbkdf2_sha256)
- **Файловое хранилище**: локальная файловая система (volume)

## 🚀 Быстрый старт

### Предварительные требования

- Docker и Docker Compose установлены на вашей машине.
- Git (для клонирования репозитория).

### 1. Клонирование репозитория

```bash
git clone https://github.com/kyberlox/TaskManager
cd TaskManager
```

### 2. Настройка окружения

Создайте файл `.env` на основе примера:

```bash
cp .env.example .env
```

Отредактируйте `.env`, указав свои значения:

```env
DOMAIN=localhost
DATABASE_URL=postgresql://postgres:postgres@db:5432/taskmanager
SECRET_KEY=your-secret-key-change-this
```

### 3. Запуск приложения

Запустите все сервисы с помощью Docker Compose:

```bash
docker-compose up --build -d
```

Эта команда соберёт образы и запустит контейнеры в фоновом режиме.

### 4. Проверка работы

После запуска откройте браузер и перейдите по адресу указанного в **.env** домена.

Если всё настроено правильно, вы увидите страницу входа.

### 5. Создание первого пользователя

1. Нажмите «Регистрация» и заполните форму.
2. После регистрации вы автоматически войдёте в систему.
3. Перейдите в профиль (верхний правый угол) для загрузки аватарки.

### 6. Создание задач и категорий

- В меню выберите «Категории» для создания категорий.
- На главной странице нажмите «Создать задачу».
- Внутри задачи можно создавать подзадачи, отправлять сообщения, загружать файлы.

## 🔧 Администрирование

Пользователи с флагом `is_admin=True` видят в профиле ссылку «Все пользователи», где могут просматривать и редактировать других пользователей.

Чтобы сделать пользователя администратором, выполните SQL-запрос в БД:

```sql
UPDATE users SET is_admin = TRUE WHERE login = 'ваш_логин';
```

Или используйте интерфейс, если уже есть хотя бы один администратор.

## 📁 Загрузка файлов

Все загруженные файлы сохраняются в директории `uploads/` внутри volume. Nginx настроен на раздачу этих файлов по URL `/uploads/`.

Максимальный размер загружаемого файла – 100 МБ.

Поддерживаемые форматы для предпросмотра:

- **Изображения**: JPEG, PNG, GIF, SVG, WebP
- **Видео**: MP4, WebM, OGG
- **Документы**: PDF (через iframe)
- **Таблицы**: Excel (.xlsx, .xls, .ods) – преобразуются в HTML-таблицу
- **Текстовые файлы**: TXT, JSON, CSV, XML – отображаются как текст

## 🗓 Календарь

На главной странице интегрирован календарь FullCalendar. Он отображает все задачи (включая неактивные) по дате выполнения. Для переключения между видами (месяц, неделя, день) используйте кнопки в заголовке календаря.

## 🤖 Виртуальный помощник (GigaChat)

Приложение включает интеграцию с GigaChat API от SberBank AI, что позволяет создавать умных помощников для задач. Помощники могут:
- Отвечать на вопросы по управлению задачами.
- Анализировать прикреплённые файлы (текст, таблицы, изображения).
- Генерировать код, изображения, тексты (если выбран соответствующий режим).
- Выполнять function calling (вызов функций) для автоматизации действий.

### Настройка GigaChat API

1. **Получите ключи API**:
   - Зарегистрируйтесь на [SberBank AI Studio](https://developers.sber.ru/studio).
   - Создайте приложение и получите `client_id` и `authorization_key`.

2. **Настройте переменные окружения**:
   В файле `.env` добавьте следующие строки (или отредактируйте существующие):
   ```env
   GIGACHAT_CLIENT_ID=ваш_client_id
   GIGACHAT_SCOPE=GIGACHAT_API_PERS
   GIGACHAT_AUTHORIZATION_KEY=ваш_authorization_key
   GIGACHAT_API_URL=https://gigachat.devices.sberbank.ru/api/v1
   GIGACHAT_MODEL=GigaChat-Lite
   ```

3. **Перезапустите приложение**:
   ```bash
   docker-compose down
   docker-compose up --build -d
   ```

4. **Создайте помощника**:
   - Войдите как администратор.
   - Перейдите в раздел «Помощники» и создайте нового помощника.
   - Укажите имя, описание, контекст, выберите возможности (генерация кода, изображений и т.д.).
   - Настройте функции (function calling) при необходимости.

5. **Используйте помощника в задаче**:
   - Откройте любую задачу.
   - В блоке «Помощник» выберите созданного помощника, введите сообщение и прикрепите файл (опционально).
   - Нажмите «Отправить» – помощник проанализирует запрос и даст ответ.

### Проблемы с генерацией изображений

При использовании функции `generate_image` может возникнуть ошибка **403 Forbidden**. Это связано с ограничениями тарифного плана GigaChat-Lite – генерация изображений через отдельный эндпоинт `/images/generations` может быть недоступна.

**Альтернативный подход**: GigaChat может генерировать изображения через чат, возвращая `file_id` в ответе. В текущей реализации используется отдельный HTTP-запрос, что может не поддерживаться вашим тарифом.

**Решение**:
1. Убедитесь, что ваш аккаунт имеет доступ к генерации изображений (тариф GigaChat Pro или выше).
2. Проверьте, что `GIGACHAT_SCOPE` установлен в `GIGACHAT_API_PERS` (персональный доступ).
3. Если ошибка сохраняется, можно отключить функцию генерации изображений у помощника или использовать демо-режим, который имитирует генерацию.

Для более глубокой интеграции рекомендуется изучить официальную документацию GigaChat API и обновить код в `app/gigachat_client.py` в соответствии с актуальными методами SDK.

### Демо-режим

Если ключи GigaChat не указаны, помощник работает в демо-режиме:
- Отвечает на простые вопросы (приветствие, помощь по задачам).
- Анализирует прикреплённые файлы (выводит содержимое текстовых файлов).
- Даёт инструкцию по настройке GigaChat API.

Демо-режим предназначен для тестирования интерфейса без реального API.

## 🔒 Безопасность

- Пароли хешируются с использованием алгоритма pbkdf2_sha256.
- Сессии хранятся в куках с установкой `HttpOnly` и `Secure` (в продакшене).
- CSRF-токены защищают формы от межсайтовой подделки запросов.
- Доступ к маршрутам требует аутентификации (кроме страниц входа/регистрации).

## 📝 Дополнительные настройки

### Изменение портов

По умолчанию приложение доступно на порту 80 (Nginx). Если нужно изменить, отредактируйте `docker-compose.yaml`:

```yaml
services:
  nginx:
    ports:
      - "80:80"
```

### Добавление SSL (HTTPS)

1. Поместите сертификаты в папку `nginx/certs/`.
2. Обновите `nginx/nginx.conf` для поддержки HTTPS.
3. Пересоберите контейнеры.

### Резервное копирование базы данных

```bash
docker-compose exec db pg_dump -U postgres taskmanager > backup.sql
```
